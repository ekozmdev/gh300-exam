## 問1（単一選択）

Copilot Spaces を使用できるユーザーはどれか？

- A) Copilot Business または Copilot Enterprise ライセンスを持つユーザーのみ
- B) Copilot ライセンス（Copilot Free を含む）を持つすべてのユーザー
- C) GitHub Enterprise Cloud の管理者のみ
- D) GitHub Pro または Team プランのユーザーのみ

**正解: B**

> 解説: ドキュメントには「Copilot のライセンス (Copilot Free を含む) を持つすべてのユーザーは、スペースを作成して使用できます」と明記されている。無料プランユーザーも利用可能だが、使用量は毎月のチャット制限にカウントされる。

## 問2（単一選択）

Copilot Space を新規作成する際のエントリーポイント（URL）はどれか？

- A) `https://github.com/features/copilot`
- B) `https://copilot.github.com/spaces`
- C) `https://github.com/copilot/spaces`
- D) `https://github.com/settings/copilot`

**正解: C**

> 解説: 公式ドキュメントでは、スペースを作成するには `https://github.com/copilot/spaces` にアクセスして「Create space」をクリックすることが案内されている。

- B) 誰でも編集できる
- C) 組織メンバーのみ閲覧できる
- D) リンクを知っているユーザーは編集できる

**正解: A**

## 問4（単一選択）

組織所有の Space における「Editor（エディター）」ロールでできないことはどれか？

- A) Space の共有設定を変更する
- B) Space の名前や説明を更新する
- C) Space に質問を送信して回答を得る
- D) Space に添付ファイルを追加する

**正解: A**

> 解説: エディターは名前・説明・添付ファイル・インストラクションの更新が可能だが、共有設定の変更や Space の削除はできない。それらはアドミン（Admin）ロールのみが実行できる。

## 問5（単一選択）

Space にリポジトリを添付した場合、Copilot はどのようにそのコンテンツを扱うか？

- A) リポジトリを検索し、質問に回答するために最も関連性の高いコンテンツのみを取得する
- B) リポジトリ全体をメモリに読み込み、すべてのクエリに使用する
- C) リポジトリの README のみを参照する
- D) デフォルトブランチの最新コミットのみを参照する

**正解: A**

> 解説: ドキュメントによれば、リポジトリを添付した場合 Copilot はプロジェクト全体をメモリに読み込むのではなく、検索して関連コンテンツのみを取得する。これは大規模リポジトリの質問応答に適した方式。

## 問6（単一選択）

Space に個別ファイルを添付した場合の動作として正しいのはどれか？

- A) ファイルの完全な内容がコンテキストウィンドウに読み込まれ、すべてのクエリに使用される
- B) ファイルの要約のみが Copilot のコンテキストに追加される
- C) ファイルはインデックス化されるが、参照は任意
- D) ファイルの最初の 1000 文字のみが使用される

**正解: A**

> 解説: 個別ファイルを添付すると、ファイルの内容全体がコンテキストウィンドウに読み込まれ、その Space のすべてのクエリで使用される。特定のドキュメントを Copilot に常に参照させたい場合に適している。

## 問7（単一選択）

Space の「Instructions（インストラクション）」コンテキストタイプの目的はどれか？

- A) Copilot の学習データを追加する
- B) Copilot が使用する SQL クエリのテンプレート
- C) スペースに追加するリポジトリの URL リスト
- D) Copilot が質問に答える際に集中すべき内容・得意分野・避けるべき内容を記述するフリーテキスト

**正解: D**

> 解説: Instructions はフリーテキストで、Copilot が Space 内で何に注力すべきか、どのようなタスクを支援すべきか、何を避けるべきかを記述するもの。これは Sources とは異なり、Copilot の応答の「方向性」を制御する。

Space の説明（description）を追加することで得られる効果はどれか？

- C) 検索エンジンのインデックスに使用される
- D) 他のユーザーが Space のコンテキストを理解するのに役立つが、Copilot の応答には影響しない

## 問9（単一選択）

Space 内で送信した質問は、Copilot Free ユーザーの場合どのようにカウントされるか？

## 問15（単一選択）

GitHub Spark のマネージドデータストアのバックエンドはどれか？

- A) GitHub の内部 PostgreSQL
- B) Firebase Realtime Database
- C) Azure Cosmos DB（マネージドキーバリューストア）
- D) Amazon DynamoDB

**正解: C**

## 問22（単一選択）

GitHub Spark を「read-only」として公開した場合の効果はどれか？

- A) アプリのコードが他のユーザーから閲覧できなくなる

- B) リポジトリから変更のみ、Spark に反映される（一方向同期）

## 問33（複数選択）

- E) タスクに必要なものを 1 か所に集めてフローを維持できる
