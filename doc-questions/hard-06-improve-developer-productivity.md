# GH-300 難問クイズ Section 6: Improve Developer Productivity
# （コード生成 / テスト / デバッグ / セキュリティ改善 / ドキュメント生成）

> 開発者生産性向上のための Copilot 活用に関する難問クイズです。
> 単一選択では「2択まで絞れるが最後の1択に悩む」問題を中心に構成しています。
> 出典: [GH-300 Study Guide (Microsoft Learn)](https://learn.microsoft.com/en-us/credentials/certifications/resources/study-guides/gh-300)

---

## 単一選択

### 問1
**IDE 上の Copilot** でテストを生成する際の最善のアプローチとして正しいのはどれですか。

- A) Copilot にコード全体のテストを一度に生成させ、そのままテストスイートに追加する
- B) Copilot が生成したテストのロジックと期待値を人間がレビューし、必要に応じて修正してから使用する
- C) Copilot はテストの生成には対応しておらず、コード補完のみに使用できる
- D) Copilot が生成したテストは常に正確であり、カバレッジも保証される


### 問2
**IDE 上の Copilot** でデバッグを行う際の活用方法として最も正確なのはどれですか。

- A) Copilot はコードの実行を直接行ってバグを特定できるため、ローカルでのデバッグが不要になる
- B) エラーメッセージやスタックトレースをコンテキストとして提供することで、Copilot から修正案や原因の説明を得られる
- C) `/fix` スラッシュコマンドを実行すると、バグのあるコードを完全に修正した状態で保存される
- D) Copilot はコンパイルエラーのみを修正でき、論理エラーには対応していない


### 問3
**Copilot のセキュリティ改善支援**に関する説明として最も正確なのはどれですか。

- A) Copilot に `/fix security` と入力すると、コードベース全体のセキュリティ脆弱性が自動修正される
- B) Copilot が生成するコードはセキュリティレビューを経ているため、脆弱性が含まれることはない
- C) セキュリティ改善には Copilot は補助的に使用できるが、検出精度が低いため主な用途はドキュメント生成に限られる
- D) Copilot は脆弱性コードのパターンを認識し修正案を提案できるが、すべての脆弱性を検出できるとは限らない


### 問4
**IDE 上の Copilot** でコードのリファクタリングを行う際の正しい理解はどれですか。

- A) Copilot はリファクタリング提案を行えるが、その提案が既存のテストを壊さないかは人間が確認する必要がある
- B) Copilot のリファクタリング提案は常にパフォーマンスの最適化を優先する
- C) リファクタリングは `/refactor` コマンド専用であり、チャットでは依頼できない
- D) Copilot がリファクタリングしたコードは自動的にテストされ、問題がなければコミットされる


### 問5
**IDE 上の Copilot** でドキュメント（コメント・docstring）を生成する際の正しい理解はどれですか。

- A) Copilot が生成したドキュメントは常に正確であり、そのまま使用して問題ない
- B) Copilot が生成したドキュメントはレビューが必要であり、コードの実際の動作と一致しているか確認する必要がある
- C) ドキュメント生成は Business プランと Enterprise プランでのみ利用できる機能である
- D) Copilot はドキュメントの生成はできるが、既存のドキュメントの更新には対応していない

## 複数選択

### 問6（3つ選択）
**IDE 上の Copilot** を使ったテスト作成において、人間がレビューすべき項目として正しいものを **3 つ**選んでください。

- A) テストケースの期待値（expected values）が正しいか
- B) エッジケースがカバーされているか
- C) Copilot が使用した IDE バージョンが適切か
- D) テストロジックがコードの実際の仕様を正しく反映しているか


### 問7（2つ選択）
**IDE 上の Copilot** でコードの説明を求める際に効果的なアプローチを **2 つ**選んでください。

- A) 説明が必要なコードブロックを選択して `/explain` を使用する
- B) コードベース全体を一度に説明させる
- C) エラーや不明な箇所を含むコードを選択してチャットに貼り付け、具体的な質問をする
- D) コメントなしのコードブロックのみを選択する（コメントがあると精度が下がるため）


### 問8（2つ選択）
**Copilot を使ったコードレビュー支援**において正しいものを **2 つ**選んでください。

- A) Copilot のレビュー提案はバグや改善点の発見に役立つが、最終的な判断は人間が行う
- B) Copilot が承認したコードは自動的にマージされる
- C) Copilot Code Review は PR の変更内容に対してコメントを自動生成できる
- D) Copilot を使ったコードレビューでは、リポジトリへのアクセス権が不要である


### 問9（3つ選択）
**開発ライフサイクルの各フェーズ**における Copilot の活用方法として正しいものを **3 つ**選んでください。

- A) コーディングフェーズ: コード補完や関数の生成
- B) テストフェーズ: ユニットテストの生成と提案
- C) 要件定義フェーズ: ステークホルダーとの会議を自動で記録・要約する
- D) デプロイフェーズ: CI/CD パイプラインのスクリプト生成のサポート


### 問10（2つ選択）
**IDE 上の Copilot** でコードの翻訳（言語変換）を行う際の正しい理解を **2 つ**選んでください。

- A) Copilot はあるプログラミング言語から別の言語へのコード変換を提案できる
- B) 変換後のコードが元のロジックを正しく再現しているか、人間が検証する必要がある
- C) Copilot は言語変換に完全対応しており、変換後のコードは必ずバグなしで動作する
- D) 言語変換はサーバー側ツールにのみ対応し、IDE ではサポートされていない
