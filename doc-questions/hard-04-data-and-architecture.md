# GH-300 難問クイズ Section 4: Data and Architecture
# （プロンプト構築 / コンテキスト / LLM の限界 / データフロー）

> Copilot 内部のデータ処理・アーキテクチャに関する難問クイズです。
> 単一選択では「2択まで絞れるが最後の1択に悩む」問題を中心に構成しています。

## 単一選択

### 問1
GitHub Copilot が IDE でコード補完を生成する際に**プロンプトを構築する**プロセスとして最も正確なのはどれですか。

- A) ユーザーが入力したコード全文を LLM に直接送信する
- B) カーソル周辺のコンテキスト（開いているファイルの内容、関連ファイル等）を収集・選別し、トークン制限内に収まるようプロンプトを組み立ててから LLM に送信する
- C) LLM がリポジトリ全体をリアルタイムにクロールして補完候補を生成する
- D) キーストロークパターンを分析して、コンテキストなしで補完候補を生成する


### 問2
Copilot の LLM において **知識のカットオフ（training cutoff）** が実務に与える影響として最も正確なのはどれですか。

- A) カットオフ以降はすべてのコード生成機能が自動的に無効になる
- B) カットオフはプロンプトの長さを制限するパラメーターであり、知識とは無関係である
- C) コンテキストウィンドウのサイズを超えた入力を切り捨てる動作であり、学習データの範囲とは別の概念である
- D) カットオフ以降に登場したライブラリやフレームワークの機能については、正確な補完が保証されない


### 問3
Copilot が IDE でコードを生成する際の**コンテキストウィンドウ（context window）** に関する説明として最も正確なのはどれですか。

- A) コンテキストウィンドウは LLM が一度に処理できるトークン数の上限を定義し、この範囲を超えたコードはプロンプトから除外される
- B) コンテキストウィンドウはユーザーセッション中の会話履歴を無制限に保持する仕組みである
- C) コンテキストウィンドウはリポジトリ全体のファイル数に対する制限である
- D) コンテキストウィンドウが大きいほど常に精度が下がる


### 問4
**IDE 上の Copilot** がコンテキストとして利用する情報の優先度について、最も正確な説明はどれですか。

- A) カーソル直前のコードのみを参照し、他のファイルは一切参照しない
- B) 開いているタブのファイル、カーソル周辺のコード、および `.github/copilot-instructions.md` などのコンテキストファイルを組み合わせて利用する
- C) `.github/copilot-instructions.md` が存在しない場合、コード補完機能は動作しない
- D) コンテキストは常にランダムに選ばれるため、精度に影響を与えない


### 問5
GitHub Copilot の**「幻覚（hallucination）」** に関する説明として最も正確なのはどれですか。

- A) 幻覚はバグのあるコードを入力した場合のみ発生する
- B) LLM は事実と異なるコードやライブラリ API を自信を持って生成することがあり、生成結果を常に検証する必要がある
- C) GitHub が実施している独自の幻覚防止フィルターにより、Copilot の出力に幻覚は含まれない
- D) 幻覚はコメントの生成でのみ発生し、コード補完では発生しない

## 複数選択

### 問6（3つ選択）
Copilot が IDE でプロンプトを構築する際に**コンテキストとして参照する可能性がある情報**を **3 つ**選んでください。

- A) 現在のファイルのカーソル前後のコード
- B) ユーザーがアクセスしたすべての GitHub リポジトリ
- C) 同じ IDE セッションで開いている関連ファイル（タブ）の内容
- D) `.github/copilot-instructions.md` の内容


### 問7（2つ選択）
**LLM の確率的な性質（stochastic nature）** がコード生成に与える影響として正しいものを **2 つ**選んでください。

- A) 同じプロンプトに対して、実行のたびに異なる出力が生成されることがある
- B) 生成されたコードは常に deterministic（決定論的）で再現性が保証される
- C) 生成されたコードには誤りが含まれる場合があり、人間によるレビューが必要
- D) 確率的な性質はプロンプトの長さにのみ依存する


### 問8（2つ選択）
Copilot の利用において**コンテキストの質を高める方法**として正しいものを **2 つ**選んでください。

- A) 関連するファイルを IDE の同じセッションで開いておく
- B) できるだけ短い（1〜2 単語の）プロンプトを使う
- C) `.github/copilot-instructions.md` にプロジェクト固有のルールや背景情報を記述する
- D) コード補完のたびに IDE を再起動する


### 問9（3つ選択）
**Copilot のデータ処理に関する説明**として正しいものを **3 つ**選んでください。

- A) コード補完のリクエストは GitHub のサーバーに送信される
- B) GitHub Copilot Business / Enterprise では、送信されたプロンプトをモデル学習に使用しない
- C) すべてのプランで、全ユーザーのプロンプトが常に LLM の再学習に使用される
- D) 生成されたコードは GitHub のサービス品質向上のために一時的に保持される場合がある


### 問10（2つ選択）
**プロキシ（GitHub Copilot proxy）** の役割として正しいものを **2 つ**選んでください。

- A) クライアントからのリクエストを受け取り、適切な LLM エンドポイントに転送する
- B) 生成されたコードを自動でリポジトリにコミットする
- C) コンテンツフィルタリング（有害コンテンツの検出など）を担う役割を持つ
- D) ユーザーのローカル環境でのみ動作し、GitHub のサーバーとは通信しない
