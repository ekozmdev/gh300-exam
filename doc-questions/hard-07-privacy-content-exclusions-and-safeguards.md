# GH-300 難問クイズ Section 7: Privacy, Content Exclusions, and Safeguards
# （コンテンツ除外 / セーフガード / REST API / データプライバシー）

> プライバシーとコンテンツ除外・セーフガードに関する難問クイズです。
> 単一選択では「2択まで絞れるが最後の1択に悩む」問題を中心に構成しています。
> 出典: [GH-300 Study Guide (Microsoft Learn)](https://learn.microsoft.com/en-us/credentials/certifications/resources/study-guides/gh-300)

---

## 単一選択

### 問1
**コンテンツ除外（content exclusions）** を設定した場合の動作として最も正確なのはどれですか。

- A) 除外されたファイルはリポジトリから完全に削除される
- B) 除外されたファイルの内容は Copilot のコンテキストから除外され、コード補完やチャットの参照対象にならない
- C) 除外設定を行うと、除外ファイルへのアクセスが組織のすべてのメンバーに対して禁止される
- D) 除外されたファイルは IDE のエディターには表示されなくなる


### 問2
**コンテンツ除外の設定場所**として正しいのはどれですか。

- A) IDE の設定画面からのみ設定でき、GitHub.com からは変更できない
- B) リポジトリ設定（Settings）または Organization 設定から設定できる
- C) `.github/copilot-exclusions.json` ファイルに記述する
- D) GitHub Enterprise の課金管理画面からのみ設定できる


### 問3
**Copilot のセーフガード（safeguards）** に関する説明として最も正確なのはどれですか。

- A) セーフガードにより、Copilot はすべての有害・不適切なコンテンツを 100% の精度でフィルタリングできる
- B) セーフガードは有害なコンテンツや脆弱性のあるコードパターンの生成を抑制するよう設計されているが、完全ではない
- C) セーフガードは GitHub Enterprise プランのみで有効になる機能である
- D) セーフガードはユーザーが任意に無効化でき、無効化すると補完精度が向上する


### 問4
**GitHub Copilot Business / Enterprise** において、ユーザーのプロンプトがモデルの追加学習に使用されるかどうかに関する説明として最も正確なのはどれですか。

- A) すべてのプランで、送信されたプロンプトはデフォルトでモデルの学習に使用される
- B) Business / Enterprise では、送信されたプロンプトは GitHub の AI モデルの学習に使用されない
- C) Business / Enterprise でも、ユーザーがオプトアウトしない限りプロンプトは学習に使用される
- D) Business / Enterprise のプロンプトはサードパーティ LLM ベンダーに送信されるため、GitHub 側では制御できない


### 問5
**Copilot の REST API** でできることとして最も正確なのはどれですか。

- A) REST API を使うと、プログラムから直接コード補完リクエストを送信してテキストを生成できる
- B) REST API は管理操作には使えず、GitHub Actions の Copilot 拡張機能を通じた利用のみに対応する
- C) REST API は Enterprise プランのみで使用でき、Business プランの組織は対象外である
- D) REST API を使って組織の Copilot シートの割り当て・解除や利用状況の取得ができる

## 複数選択

### 問6（2つ選択）
**コンテンツ除外（content exclusions）** の設定において正しいものを **2 つ**選んでください。

- A) 除外設定は Organization レベルで行い、組織内リポジトリ全体に適用できる
- B) 除外設定はリポジトリ設定で行い、特定リポジトリのみに適用できる
- C) 除外設定をしたファイルはすべての Git 操作も禁止される
- D) 除外設定はユーザーが任意に上書きして解除できる


### 問7（3つ選択）
**GitHub Copilot のプライバシー設定**に関して正しいものを **3 つ**選んでください。

- A) GitHub Copilot Individual では、デフォルトでプロンプトを学習に使用しない設定に変更できる
- B) Business / Enterprise では、プロンプトデータはモデルの学習に使用されない
- C) すべてのプランで、ユーザーのコードは GitHub の社員が常時モニタリングしている
- D) コンテンツ除外を設定することで、機密ファイルを Copilot のコンテキストから外せる


### 問8（2つ選択）
**Copilot のコンテンツフィルタリング（content filtering）** に関して正しいものを **2 つ**選んでください。

- A) 有害なコンテンツの生成を抑制するためにコンテンツフィルターが動作する
- B) コンテンツフィルターは 100% の精度で有害コンテンツを除去できる
- C) フィルタリングの結果、Copilot の応答が一部ブロックされることがある
- D) コンテンツフィルターはユーザー側でカスタマイズして無効にできる


### 問9（3つ選択）
**Copilot の利用を監査するための仕組み**として正しいものを **3 つ**選んでください。

- A) Organization の監査ログ（audit log）でポリシー変更を追跡できる
- B) GitHub の監査ログに、Copilot が生成した補完コード全文が記録される
- C) REST API を使って組織の Copilot 利用状況を取得できる
- D) GitHub.com の Organization 設定画面で Copilot 利用状況の概要を確認できる


### 問10（2つ選択）
**IP インデムニティ（知的財産補償）** に関するものとして正しいものを **2 つ**選んでください。

- A) GitHub Copilot Enterprise / Business では、IP インデムニティ（著作権侵害に対する法的保護）が提供される場合がある
- B) IP インデムニティはすべてのプランで無条件に提供される
- C) コンテンツフィルタリングで「一致するコードのブロック（duplication detection）」を有効にすることで、公開コードとの一致を低減できる
- D) IP インデムニティが適用されると、生成コードの品質が向上する
