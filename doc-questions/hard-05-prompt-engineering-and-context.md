# GH-300 難問クイズ Section 5: Prompt Engineering and Context

## 単一選択

### 問1
**few-shot プロンプティング（few-shot prompting）** の説明として最も正確なのはどれですか。

- A) プロンプトに指示のみを含め、例を一切提供しない手法（zero-shot ともいう）
- B) 大量のトレーニングデータを事前に LLM に投入してファインチューニングする手法
- C) 1 つのプロンプトで複数の LLM を並列呼び出しし、最良の出力を選択する手法
- D) プロンプトに入出力の例を複数含めることで、モデルに期待する出力パターンを示す手法


### 問2
**`.github/copilot-instructions.md`** と **`.github/instructions/*.instructions.md`** の違いとして最も正確なのはどれですか。

- A) `.github/copilot-instructions.md` はすべての Copilot Chat セッションに常に適用される。一方、`.github/instructions/*.instructions.md` はファイルマッチングや明示的な参照によって特定のコンテキストで使用される
- B) 両ファイルは機能が同一であり、どちらを使用しても結果に差はない
- C) `.github/instructions/*.instructions.md` はすべてのセッションに常に適用され、`.github/copilot-instructions.md` はオプトイン形式である
- D) `.github/copilot-instructions.md` は IDE のコード補完にのみ適用され、チャットには影響しない


### 問3
**カスタム命令（custom instructions）の優先順位**として最も正確なのはどれですか。

- A) Organization > Repository > Personal の順で、組織の設定が最優先される
- B) Personal > Repository > Organization の順で、個人設定が最優先される
- C) Repository > Personal > Organization の順で、リポジトリ設定が最優先される
- D) すべての設定は同等の優先度で扱われ、競合時は後から読み込まれたものが優先される


### 問4
**IDE 上の Copilot** で効果的なプロンプトを書くためのベストプラクティスとして最も正確なのはどれですか。

- A) プロンプトは常に短くすべきであり、詳細な説明は精度を下げる
- B) 具体的な期待出力・制約条件・コンテキストを含む詳細なプロンプトを作成することで、より正確な補完が得られる
- C) プログラミング言語を指定してはならず、Copilot に言語を推定させる方が精度が高い
- D) プロンプトは英語のみで記述すべきであり、他の言語では機能が制限される


### 問5
**`@workspace` コンテキスト変数**（VS Code 上の Copilot Chat）の説明として最も正確なのはどれですか。

- A) `@workspace` を指定すると、GitHub リポジトリ全体の履歴（コミット・PR・Issue）を検索する
- B) `@workspace` を指定すると、Copilot がワークスペース内のコードを検索・参照しながら応答する
- C) `@workspace` は GitHub.com 専用の機能であり、VS Code では使用できない
- D) `@workspace` を指定すると、外部 MCP サーバーへの接続が自動で有効になる

## 複数選択

### 問6（3つ選択）
**効果的なプロンプトの特徴**として正しいものを **3 つ**選んでください。

- A) 具体的で明確な指示を含む
- B) 期待する出力の形式や制約を指定する
- C) プロンプトは必ず 1 行以内に収める
- D) 十分なコンテキスト（背景・目的）を提供する


### 問7（2つ選択）
**Copilot Chat の `#` 記法（チャット変数）** でコンテキストとして指定できるものとして正しいものを **2 つ**選んでください（VS Code を想定）。

- A) `#file` で特定のファイルを参照させる
- B) `#time` で現在の日時情報を取得する
- C) `#selection` でエディターで選択中のコードを参照させる
- D) `#network` でネットワーク設定を参照させる


### 問8（2つ選択）
**zero-shot プロンプティング** に関する説明として正しいものを **2 つ**選んでください。

- A) 例を提供せずに、タスクの説明のみでモデルに出力を生成させる手法
- B) モデルの学習データに存在しないタスクには全く対応できない
- C) シンプルなタスクや、モデルが十分な事前知識を持つタスクで有効
- D) zero-shot では必ず few-shot より精度が高くなる


### 問9（3つ選択）
**Copilot のコンテキスト管理**において、コンテキストの質と量に関する正しい説明を **3 つ**選んでください。

- A) トークン制限を超えたコンテキストは切り捨てられ、プロンプトに含まれない
- B) 関連度の低いファイルを多数開くことはコンテキストの質を下げる可能性がある
- C) コンテキストが多いほど常に出力精度が向上する
- D) 適切なコンテキストを提供することで補完の精度が向上する


### 問10（2つ選択）
**VS Code での Copilot Chat における `@` メンション（チャット参加者）** として正しいものを **2 つ**選んでください。

- A) `@workspace` でワークスペース内のコードを検索・参照する
- B) `@terminal` でターミナルに関する質問を行う
- C) `@database` でデータベーススキーマを自動取得する
- D) `@browser` で現在表示中の Web ページを解析する
