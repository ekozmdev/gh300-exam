# GH-300 Section 6 クイズ: Improve Developer Productivity with GitHub Copilot

対象は `Section 6: Improve Developer Productivity with GitHub Copilot` です。  
単一選択 5 問、複数選択 5 問で構成しています。

## 問題

### 問1 [単一選択]
GH-300 のこのセクションで、GitHub Copilot が目指す効果として最も適切なものはどれですか。

- A. 開発の生産性、品質、セキュリティを高める
- B. リポジトリの公開範囲を自動で変更する
- C. Issue を自動でクローズする
- D. ブランチ保護を無効化する

解答: A
解説: `What is GitHub Copilot?` では、Copilot は開発の productivity, quality, security を高めるためのものだと説明されています。
参考: [GitHub Copilot とは](https://docs.github.com/ja/copilot/get-started/what-is-github-copilot)

### 問2 [単一選択]
GitHub Copilot の機能として、最も直接的にこのセクションの説明に合うものはどれですか。

- A. リポジトリの監査ログを可視化する
- B. Organization の所有者を自動で付け替える
- C. コード生成、リファクタリング、ドキュメント作成を支援する
- D. Git タグを自動で再命名する

解答: C
解説: `GitHub Copilot features` では、コード生成、リファクタリング、ドキュメント作成への活用が明示されています。
参考: [GitHub Copilot の機能](https://docs.github.com/ja/copilot/get-started/features)

- A. 依存関係を削除してコードを短くする
- B. 学習の加速とコンテキストスイッチの削減に役立てる
- C. すべてのテストをスキップする
- D. 既存の PR を自動でマージする

解説: `GitHub Copilot features` では、学習の加速とコンテキストスイッチ削減が用途として挙げられています。

### 問4 [単一選択]
GitHub Copilot CLI の説明として最も適切なのはどれですか。

- A. GitHub Actions の YAML を生成する専用画面
- B. Pull Request を手動で承認するためのダッシュボード
- C. セキュリティ警告だけを表示する監視ツール

解説: `GitHub Copilot CLI` の説明は、ターミナルから直接 Copilot を使う点が中心です。
