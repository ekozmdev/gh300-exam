# GH-300 Section 1 クイズ: Use GitHub Copilot Responsibly

対象ドキュメント:
- [GitHub での GitHub Copilot Chat の責任ある使用](https://docs.github.com/ja/copilot/responsible-use/chat-in-github)
- [GitHub Copilot コード レビューの責任ある使用](https://docs.github.com/ja/copilot/responsible-use/code-review)
- [GitHub Copilot を使ったコードレビュー](https://docs.github.com/ja/copilot/how-tos/use-copilot-agents/request-a-code-review/use-code-review)

## 単一選択

### 問1
GitHub Copilot code review の位置づけとして最も適切なものはどれですか。

- A) 人間のレビューを完全に置き換える機能
- B) リポジトリの課金設定を変更する機能
- C) pull request を自動でマージする機能
- D) 人間のレビューを補完する機能


### 問2
GitHub 上の Copilot Chat の主な目的として正しいものはどれですか。

- A) 組織の課金を変更する
- B) 既定ブランチ名を一括変更する
- C) コーディング関連の質問に答える
- D) 2 要素認証を無効化する


### 問3
GitHub Copilot code review について正しいものはどれですか。

- A) モデルをレビューごとに自由に切り替えられる
- B) 1 回使っても Premium request は消費しない
- C) すべての問題を必ず検出する
- D) モデルの切り替えはサポートされていない


### 問4
Copilot code review を使う際に注意すべきリスクとして最も適切なものはどれですか。

- A) 必ずリポジトリの可視性が変更される
- B) 依存関係グラフが削除される
- C) 存在しない問題を指摘する誤検知があり得る
- D) すべてのコメントは手動で作成される


### 問5
Copilot に問題があるレビュー コメントがあった場合、推奨される行動として最も適切なものはどれですか。

- A) 何もしない
- B) サムアップまたはサムダウンでフィードバックを返す
- C) すぐに全 PR をクローズする
- D) 直ちに Copilot を無効化する


## 複数選択

### 問6
責任ある Copilot 利用として正しいものを 3 つ選んでください。

- A) 提案は常に正しいとみなす
- B) 人間のレビューで補完する
- C) 生成結果を検証する
- D) 問題があればフィードバックを返す


### 問7
GitHub 上の Copilot Chat について正しいものを 3 つ選んでください。

- A) 非コーディング用途が主目的である
- B) リポジトリデータなどの追加コンテキストを使って回答できる
- C) コードの説明や生成に使える
- D) コーディング関連の質問向けである



### 問8
Copilot code review の入力や応答について正しいものを 3 つ選んでください。

- A) 常に監査ログだけを入力として使う
- B) pull request のタイトルや本文も入力に使われることがある
- C) 応答には自然言語の提案やコード提案が含まれうる
- D) コード変更と関連コンテキストからプロンプトが作られる


### 問9
Copilot code review の制限として正しいものを 3 つ選んでください。

- A) すべての問題を見つけられるとは限らない
- B) 誤検知が起こり得る
- C) 常に人間より正確である
- D) 提案コードが不正確または安全でない場合がある


### 問10
カスタム命令と Copilot code review の関係として正しいものを 2 つ選んでください。

- A) カスタム命令ファイルは無制限に全文読まれる
- B) カスタム命令を使うと人間のレビューは不要になる
- C) コーディングスタイルやベストプラクティスの理解に役立つ
- D) pull request レビュー時はベースブランチ側の命令が使われる
