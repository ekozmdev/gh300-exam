# GH-300 難問クイズ Section 1: Use GitHub Copilot Responsibly

> 2026年1月改訂後の新試験範囲に対応した難問クイズです。
> 単一選択では「2択まで絞れるが最後の1択に悩む」問題を中心に構成しています。
> 出典: [GH-300 Study Guide (Microsoft Learn)](https://learn.microsoft.com/en-us/credentials/certifications/resources/study-guides/gh-300)

---

## 単一選択

### 問1
GitHub Copilot を「責任を持って運用する（operate responsibly）」ことの説明として、公式の学習ガイドに最も合致するのはどれですか。

- A) AI 出力を人間がレビューし、問題があればフィードバックを返すプロセスを継続的に維持する
- B) Copilot の利用を社内で文書化されたユースケースのみに限定し、それ以外では使用しない
- C) 生成された提案はすべて、コミット前に独立した静的解析ツールで検証する
- D) Enterprise プランに契約して監査ログと重複検知を有効化し、利用状況を記録する


### 問2
AI の「公平性（Fairness）」に関わる潜在的な害として最も具体的に適切なのはどれですか。

- A) Copilot が古い API を使ったコードを提案し、後でメンテナンスコストが増加する場合
- B) 機密情報がプロンプトに意図せず含まれ、外部に送信されてしまう場合
- C) トレーニングデータの偏りにより、採用システムや与信判断など特定グループに不利なロジックが生成される場合
- D) 生成コードに既知のセキュリティ脆弱性のパターンが含まれている場合


### 問3
責任ある AI 利用における「透明性（Transparency）」の説明として最も適切なのはどれですか。

- A) Copilot の使用状況を組織全体に公開し、すべてのメンバーが確認できる状態にする
- B) 監査ログを有効化して、Copilot の利用イベントをすべて記録する
- C) GitHub がモデルのアーキテクチャと学習データの詳細を公開していること
- D) AI が生成・支援した内容であることを、チームやステークホルダーに適切に開示する


### 問4
AI 出力の検証が特に重要な理由として、生成 AI に固有の問題を最も正確に説明しているのはどれですか。

- A) LLM の知識カットオフのため、常に最新の技術情報が欠落している
- B) 出力が文法的・論理的に正しく見えても、事実として誤った内容（ハルシネーション）が含まれることがある
- C) Copilot はインターネットから直接コードをコピーするため、著作権侵害が常に起こりうる
- D) AI はユーザーの意図を完全に把握できないため、意図と異なるコードが必ず生成される


### 問5
GitHub Copilot code review で「誤検知（false positive）」があった場合の推奨対応として最も適切なのはどれですか。

- A) Copilot code review を一時的に無効化し、別のツールでレビューを再実施する
- B) Organization 管理者にエスカレーションして、コードレビューポリシーの修正を依頼する
- C) 誤検知と判断したコメントにサムダウンでフィードバックを返し、レビューを進める
- D) 誤検知はモデルの仕様であるため記録せず、PR のマージを優先する

## 複数選択

### 問6（3つ選択）
生成 AI ツールの「リスクと限界（Risks and Limitations）」として認識すべきものを **3 つ**選んでください。

- A) トレーニングデータの偏りにより、生成物に偏りが生じる可能性がある
- B) 有料プランを使用すれば、生成コードの法的リスクは GitHub が保証する
- C) LLM の知識カットオフにより、最新の技術情報が反映されない場合がある
- D) 生成コードにセキュリティ上の脆弱性が含まれる可能性がある


### 問7（2つ選択）
AI 利用による「潜在的な害を緩和する（Mitigate Harms）」方法として適切なものを **2 つ**選んでください。

- A) コンテンツ除外設定を利用して、機密情報が Copilot のコンテキストに含まれないようにする
- B) 組織内の全開発者の Copilot 利用を一時的に停止し、代替手段を検討する
- C) 生成コードに対して自動テストと人間によるコードレビューを組み合わせる
- D) 提案を受け入れる前に、複数の AI サービスで同じプロンプトを試して比較する


### 問8（2つ選択）
「倫理的かつ責任ある AI 利用（Ethical and Responsible AI Usage）」として推奨される行動を **2 つ**選んでください。

- A) Copilot が生成したコードはすべてコードコメントで明示することをチームで必須化する
- B) AI 出力の正確性と安全性を人間がレビューして確認する
- C) 問題のある提案にフィードバックを返し、AI の改善に貢献する
- D) GitHub が Copilot 提案の品質を保証しているとみなして信頼する


### 問9（3つ選択）
GitHub Copilot を「責任を持って運用する」ことに含まれる要素を **3 つ**選んでください。

- A) 提案の正確性と安全性を人間が検証する
- B) Copilot の利用コスト削減を最優先事項とする
- C) AI 出力に問題があれば積極的にフィードバックを返す
- D) Copilot の能力と限界を理解した上で利用する


### 問10（2つ選択）
ハルシネーション（AI が事実と異なる内容を確信を持って述べる現象）への適切な対策を **2 つ**選んでください。

- A) Copilot の回答を参考情報として扱い、重要な判断は自分で調べて裏付けを取る
- B) Copilot が高い確信度で答えた場合は、ハルシネーションのリスクが低いとみなして信頼する
- C) 有料プランにアップグレードすることで、ハルシネーションの発生率を大幅に下げられる
- D) 生成されたコードや説明は、実際にテストや検証を行って確認する
